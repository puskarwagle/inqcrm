## ğŸ“‹ Compliance Checklist

Use this checklist when reviewing code or generating new code:

### Before Generating Any Code
- [ ] Understand module and dependencies
- [ ] Check similar existing implementations
- [ ] Verify file goes in correct directory
- [ ] Confirm naming conventions (check QUICK_REFERENCE.md)
- [ ] Check required interfaces/contracts
- [ ] Ensure proper namespace usage

### During Code Generation
- [ ] Use type hints consistently
- [ ] Add PHPDoc comments
- [ ] Include error handling
- [ ] Follow PSR-12 standards
- [ ] Use dependency injection
- [ ] Implement database indexing
- [ ] Add input validation

### After Code Generation
- [ ] Verify imports are correct
- [ ] Check for unused imports
- [ ] Confirm method signatures match interfaces
- [ ] Validate database constraints
- [ ] Test edge cases mentally
- [ ] Document public methods

### Code Review Checklist
- [ ] PHP 8.2+ syntax (no 8.1 features)
- [ ] Laravel 10.0+ APIs (no deprecated methods)
- [ ] Pest framework for tests (not PHPUnit)
- [ ] Repository pattern for data access
- [ ] FormRequest for validation
- [ ] Policy for authorization
- [ ] Soft deletes where appropriate
- [ ] Proper indexing on foreign keys
